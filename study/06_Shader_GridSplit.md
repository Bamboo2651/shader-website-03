# Shader解説: Grid Split (空間分割とズーム)

`slide.frag` 内に記述されている `effectGridSplit` 関数についての解説です。
画面全体をいくつかの格子（グリッド）に分割し、それぞれのタイルごとに別々の拡大率やスライドを掛け合わせることで、空間がバラバラに崩れるようなエフェクトを生み出します。

## グリッド座標系への変換
画面の UV 座標 (0.0 〜 1.0) をグリッドの数（今回は 5×5）に掛け算して切り捨てると、タイルごとのID「`gridId`」が手に入ります。
```glsl
vec2 gridCount = vec2(5.0, 5.0);
vec2 gridId = floor(uv * gridCount); // (0,0), (1,0) ... (4,4) などの整数値になる
```
この `gridId` をランダム関数に渡すことで、「タイルごとに異なるランダムな値」が取得できるため、タイルがバラバラなタイミングで切り替わるようになります。

## セル（タイル）ごとのローカルズーム計算
タイルごとに中心点を求め、その中心点から UV を特定の倍率で引き伸ばす（ズームする）計算を行います。
```glsl
float cellDist = abs(mappedP - n); // 進行度とランダム値の差
float localScale = 1.0 + cellDist * 0.8 * sin(p * PI); // ズーム倍率

vec2 center = (gridId + 0.5) / gridCount; // このタイルの真ん中
vec2 zoomUv = center + (uv - center) * localScale; // 中心を基準に拡大
```

## スライド切り替え時の「ズレ（ジャンプ）」の回避
トランジションの開始時（`p = 0.0`）と終了時（`p = 1.0`）には画像がピッタリ通常の位置に収まっている必要があります。これを考慮しないと、エフェクトが終わった直後に画像が別の位置にパッと移動（ジャンプ）してしまいます。
```glsl
// t0 は p を基準にスライドアウト
vec2 slideOffset0 = vec2(0.0, p * 0.2 * dir);
// t1 は p-1 を基準にスライドイン
vec2 slideOffset1 = vec2(0.0, (p - 1.0) * 0.2 * dir);
```
このように `t0` 用のアニメーション変数と `t1` 用のアニメーション変数を完全に分けることで、バグのない美しいトランジションが実現しています。
