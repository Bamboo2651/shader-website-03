# Shader解説: Displacement (歪みと波)

`slide.frag` 内に記述されている `effectDisplacement` 関数についての解説です。
このエフェクトは、画像を「水面」のように歪ませながら、新しい画像をスライドさせて登場させます。

## アスペクト比補正 (`getCoverUV`)
前提として、画像の縦横比を崩さず（伸び縮みさせず）表示するために `getCoverUV` という関数を通しています。HTML/CSSにおける `background-size: cover;` と同じ仕組みです。

## 遅延効果（Delay）による波の表現
単に画像が切り替わるのではなく、画面の「右上から左下に向かって」徐々に切り替わっていくような演出をしています。
```glsl
// uv.y が大きい(上), uv.x が小さい(左)場所で時間がズレる
float delayValue = p * 7.0 - uv.y * 2.0 + uv.x - 2.0;
delayValue = clamp(delayValue, 0.0, 1.0);
```
この `delayValue` は、`p` (進行度)に沿って0から1へ変化しますが、画面の場所（`uv`）によって到達するタイミングが異なります。

## 正弦波での歪み (sin関数)
`sin(uTime)` を使うことで、常に一定のテンポで揺れ続ける波を作り出しています。
```glsl
vec2 w = sin(sin(uTime * 0.5) * vec2(0.0, 0.3) + uv * vec2(0.0, 4.0)) * vec2(0.0, 0.5);
```

さらに、トランジションの途中（`p=0.5`）で一番大きく歪み、最初（`p=0`）と最後（`p=1`）には歪みがゼロになるように `triP` を計算して掛け合わせています。
これにより、切り替わり途中だけ画像がフニャッと歪むオーガニックな演出を実現しています。
